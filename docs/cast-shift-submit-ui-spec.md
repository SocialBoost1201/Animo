# Cast Shift Submit UI Spec

## Purpose

キャストがスマホで迷わず、速く、ミスなくシフト提出できる画面を定義する。

この画面は以下を同時に満たすことを目的とする。

- 誰が使っても操作しやすい
- 見やすい
- モダンでプロフェッショナル
- 将来の機能追加でもUIが崩れにくい

## Core UX Principles

1. 1日ごとの入力を独立させる
2. 手入力をなくし、プルダウン中心にする
3. 営業日と休業日を一目で判別できるようにする
4. スマホで片手操作しやすいタップ領域を確保する
5. 入力済みか未入力かを視覚的に明確にする

## Scope

この仕様はキャスト側の `シフト提出` 画面に適用する。

対象:

- 月曜から土曜までの提出UI
- 祝日の休業日表示
- 出勤 / 休み切替
- 出勤時間 / 退店時間プルダウン
- 提出ボタン

対象外:

- シフト変更申請画面
- 管理者承認画面
- 公開シフト反映ロジック

## Information Architecture

画面構成は以下とする。

1. ページヘッダー
2. 対象週の表示
3. 入力ルールの短い説明
4. 月曜〜土曜の6日分カード
5. 固定CTA `シフトを提出する`

## Page Header

表示内容:

- タイトル: `来週のシフトを提出`
- 説明文:
  `出勤する日だけを選んで、開始時間と退店時間を入力してください。`
- 戻る導線:
  `ダッシュボードへ戻る`

要件:

- H1は1つのみ
- 余白に十分な呼吸感を持たせる
- ヘッダーだけで「何をする画面か」が伝わる

## Target Week Block

表示内容:

- 対象週の開始日
- 対象週の終了日
- 週送り / 週戻し

補助文:

- `月曜から土曜まで入力してください`
- `日曜と祝日は休業日です`

要件:

- 週の切替は簡単にできる
- 現在どの週を編集しているかが明確

## Daily Card Structure

1日ごとに1カードで表示する。

カード内表示要素:

- 日付
- 曜日
- 営業日 / 休業日バッジ
- `出勤する / 休む` 切替
- 出勤時間プルダウン
- 退店時間プルダウン
- 入力済み状態の視覚表示

### Card States

#### Business Day / Off

- 営業日
- `休む` が選択されている
- 時間入力欄は非表示

#### Business Day / Working

- 営業日
- `出勤する` が選択されている
- 出勤時間と退店時間のプルダウンを表示

#### Holiday

- 祝日または休業日
- `休業日` バッジ表示
- 入力欄は disabled または非表示
- 休業理由がわかるようにする

## Supported Days

### Input Target

- 月曜
- 火曜
- 水曜
- 木曜
- 金曜
- 土曜

### Non-Target

- 日曜
- 祝日

### Holiday Handling

祝日は以下の扱いにする。

- カード自体は表示する
- `休業日` と明示する
- 入力不可
- 色調を営業日と分ける

目的:

- 「存在しない」のではなく「休業日だから入力不要」と理解できるようにする

## Input Method

すべてプルダウン選択とする。

禁止:

- 自由入力
- キーボードによる時刻手打ち

理由:

- 誤入力防止
- フォーマット統一
- スマホでの入力負荷軽減

## Shift Type Toggle

各営業日に以下の切替を置く。

- `休む`
- `出勤する`

要件:

- 2択で直感的
- 選択中状態が明確
- スマホで押しやすいサイズ

## Start Time Dropdown

### Range

- 19:00 から 24:00 まで
- 30分単位

### Options

- 19:00
- 19:30
- 20:00
- 20:30
- 21:00
- 21:30
- 22:00
- 22:30
- 23:00
- 23:30
- 24:00

### Label

- `出勤時間`

## End Time Dropdown

### Range

- 21:00 から 26:00 まで
- 30分単位

### Options

- 21:00
- 21:30
- 22:00
- 22:30
- 23:00
- 23:30
- 24:00
- 24:30
- 25:00
- 25:30
- 26:00

### Label

- `退店時間`

### Note

- UI上の表示は `24:30`, `25:00`, `25:30`, `26:00` として扱う
- 内部保存形式は別途定義してよいが、表示はユーザーにわかりやすいことを優先する

## Validation Rules

1. `出勤する` を選んだ日は `出勤時間` と `退店時間` の両方が必須
2. `退店時間` は `出勤時間` より後でなければならない
3. `休む` の日は時間未入力でよい
4. 祝日は入力対象に含めない
5. 送信時に不正な時間組み合わせがないこと

## Submission CTA

ボタン文言:

- `シフトを提出する`

要件:

- 画面下部に常時わかりやすく配置
- スマホで押しやすいサイズ
- 送信中状態を表示
- 提出完了後は完了通知を出す

## Visual Design Rules

### Overall Tone

- Modern
- Professional
- Clear
- Premium but calm

### Card Design

- 1日1カード
- カードごとの境界が明確
- 余白は広め
- 情報を詰め込みすぎない

### Typography

- 日付は見つけやすく
- ラベルは補助的に
- 本文・説明は細かすぎない

### Color Usage

- 営業日: ニュートラル
- 入力済み出勤日: アクセント
- 休み: 落ち着いた薄色
- 祝日: 休業とわかる専用トーン
- エラー: 赤系で明確

## Mobile UX Rules

1. 片手でスクロールと選択ができる
2. プルダウンは押しやすい高さを確保する
3. 1カード内の情報量を増やしすぎない
4. 画面下固定CTAを採用してよい
5. フォーム全体が縦に自然に流れること

## Empty / Existing Data Rules

既存提出データがある場合:

- 対象週の入力欄に初期表示する
- 承認待ち / 承認済み / 差し戻し状態を表示する
- 再提出可能条件を明示する

## Future-Proof Structure

UIが将来崩れないよう、以下を前提にする。

- 1日カードは再利用可能な小コンポーネント化
- 時刻プルダウンの候補は定数化
- 祝日判定はロジックをUIから分離
- 状態バッジは共通コンポーネント利用
- シフト提出と変更申請のUIロジックを混ぜない

## Acceptance Criteria

以下を満たしたら完成とみなす。

1. 月曜〜土曜の入力ができる
2. 祝日は休業日としてわかる
3. 出勤時間が 19:00〜24:00 の30分単位で選べる
4. 退店時間が 21:00〜26:00 の30分単位で選べる
5. スマホで操作しやすい
6. 見た目がモダンでプロフェッショナル
7. 初見でも迷わない
