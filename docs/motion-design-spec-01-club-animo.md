# Club Animo Webサイト モーションデザイン仕様書

**ドキュメントID:** motion-design-spec-01  
**受領日時:** 2026-03-06

---

## 0. 目的

白基調の高級感を崩さず、最先端の体験としてアニメーションを全ページに実装する。  
「派手さではなく、上品さと滑らかさで記憶に残す」「表示速度を落とさず、触り心地で差を付ける」ことを目的とする。

---

## 1. 前提

- 参考サイト同等以上のモーションを使う（情報設計の一部として扱う）
- パフォーマンス優先（重い3D・派手な点滅は禁止）
- **motion-reduced環境:** 動きを抑制する
- **プレースホルダ:** 画像差し替え前でもモーションを成立させる

---

## 2. モーションの基本ルール

### 2.1 全体テンポ

- 基本はゆっくりめ。速い動きはCTAや操作反応だけに限定。
- ページ単位で変化を付け、同じ動きを乱発しない。

### 2.2 イージング

- カクつき禁止、終わりは柔らかく止める。
- 反応系（ボタン等）はキビキビ、演出系はしっとり。

### 2.3 継続性

- 全ページで同一のモーション言語を共有する（フェードとスライドの方向、テキストリビール形式の統一など）。

---

## 3. 使うモーションの種類

| 分類               | 種類                                                                  |
| ------------------ | --------------------------------------------------------------------- |
| **セクション出現** | フェードイン / 下からの軽いスライドイン / 文字リビール / 画像微ズーム |
| **リスト出現**     | スタッガー表示（キャストカード、ニュースカード、ギャラリー）          |
| **操作反応**       | ボタン押下で沈む / ホバーで微浮き / タップ時の微振動（視覚再現）      |
| **スクロール連動** | ヒーローのゆっくりズーム / 背景の軽いパララックス（大きい移動は禁止） |
| **ページ遷移**     | 画面切替はフェードとスライドの合成（主要ページは個性を付ける）        |

---

## 4. 禁止事項

- 回転の多用 / 激しい点滅 / 大きい揺れ / 過剰なパララックス
- フルスクリーン動画の自動再生で音あり / 過剰な粒子エフェクト

---

## 5. ページ別モーション仕様

### 5.1 Top (狙い: シャンデリアを象徴として刷り込む)

- **Hero:** 画像プレースホルダはロード直後にフェードイン / コピーは文字リビール / スクロールでヒーローがゆっくりズーム / 予約CTAは少し遅れて出現
- **Today Cast:** カルーセルカードはスタッガー / 出勤バッジは軽いポップ
- **Concept:** 見出しリビール / 画像ブロック微ズーム
- **Gallery Preview:** カードホバーで微浮き / タップ押下反応
- **System Preview:** 数字ブロック段階的出現
- **Night Style診断:** ステップ遷移はスライド / 各種ボタン押下沈み / 結果カードスタッガー
- **Recruit Teaser:** 2カードが左右から出現 / クリックで軽いページ遷移

### 5.2〜5.11 その他主要ページ

- **System:** 料金ケースカードはスムーズスライド / 注意事項は滑らかなアコーディオン
- **Cast一覧:** カードスタッガー / フィルタ並び替えアニメーション / 即時反映（遅延なし）
- **Cast詳細:** ギャラリースムーズトランジション / プロフィール段階表示
- **Shift:** タブ切替スライド / リストスタッガー（本日=速め・週間=ゆっくり）
- **Gallery:** 画像グリッドスタッガー / カテゴリ切替フェード / モーダル拡大（滑らかな開閉）
- **Event/News:** 一覧カードスタッガー / 詳細は見出しリビール、本文段階表示
- **Access:** Mapフェードイン / コピーボタン押下反応強め / ルート説明段階表示
- **FAQ:** アコーディオン滑らか開閉 / カテゴリ切替フェード
- **Reserve:** 入力時エラー滑らか表示 / 送信中ローディング / 完了画面フェード・スライド

### 5.12 Recruit Cast (狙い: 不安を消し最後まで読ませる)

- ストーリー型スクロール / 待遇表は行ごとのリビール / 安心安全は強調カードで段階表示 / 応募CTA追従・出し分け

### 5.13 Recruit Staff (狙い: スタッフ不足解決)

- 職種カードホバー展開 / 職種選択で仕事内容スムーズ切替 / キャリア・応募流れのステップ表示

---

## 6. コンポーネント別モーション仕様

- **Button:** hover 微浮き / active 沈み / focus 強調
- **Card:** hover 微浮き / click 押下反応 / default スタッガー出現
- **TagChip:** 選択時に強調トランジション
- **Accordion:** 高さ変化は滑らかに / アイコン回転は軽く
- **Tabs:** インジケータ滑らか移動
- **Modal:** 背景フェード / 本体スケールイン
- **Toast:** 下からスライドイン / 自動消失フェード

---

## 7. 画像プレースホルダとモーションの整合

- プレースホルダはスケルトンを必ず表示
- 画像が差し込まれた瞬間にフェードで切替
- **画像未設定でもモーションは必ず成立させること**

---

## 8. パフォーマンス基準 / 9. motion-reduced

- LCP悪化演出禁止 / ヒーロー動画は軽量・音なし・静止画フォールバック / 連続アニメーション最小限（重い場合は段階ロード）
- **motion-reduced:** 大きいスクロール停止 / 出現アニメ短縮・無し / 操作反応のみ最小限残す

---

## 10. 受け入れ基準

- 全ページにモーションが入り、最先端感がある
- 白基調の高級感が崩れていない、体験の統一感がある
- プレースホルダ状態でもレイアウト・動きが破綻しない
- 表示速度が著しく落ちていない
