# Club Animo Webサイト 要件定義書

**ドキュメントID:** requirements-01  
**受領日時:** 2026-03-06

---

## 0. このドキュメントの目的

Antigravityが参考サイトをURL解析し、Club Animoの最先端モダンかつ白基調の高級サイトを設計・実装するための要件定義。最終成果物は運用まで見据えた実務仕様とし、予約と問い合わせの成果最大化を目的とする。

---

## 1. 前提と制約

### ブランドと内装

- 内装は明るめ
- 店内に大きいシャンデリアあり（最大の売り）
- 箱は広く、高級感のある作り

### 顧客と提供価値

- 客層は富裕層が多い
- 初来店の不安を減らし、安心して予約できる情報設計が必須

### キャスト露出制約

- キャストの大半は顔出し不可
- 後ろ姿・雰囲気カット中心の掲載は可能

### 料金公開方針

- システムは同業他社と同様に公開
- ボトル価格は非公開でよい

### 運用制約

- 店長や運営が日々更新できる管理画面が必須
- 本日出勤は毎日更新される想定

## 3. ゴールとKPI

### ゴール

高級感のある世界観を担保しつつ、予約と問い合わせを最大化する

### KPI

- 予約ボタン到達率
- 電話タップ率
- 問い合わせ送信率
- キャスト詳細閲覧数
- 本日出勤からキャスト詳細への遷移率
- アクセスページ到達率

## 5. コアUI要件

### 共通

- モバイルファースト
- ヘッダー固定でCTA常設（予約 / 電話 / Instagram / 問い合わせ）
- スクロールしても迷子にならないナビ設計
- 余白とタイポで高級感を作る
- 白背景を主役にし、差し色は黒と金を最小限
- 写真トーンは明るい高級感に統一

### トップページ要件

- シャンデリアを主役にしたファーストビュー
- テキストは短く強い一文
- 主要導線は3つに絞る（本日出勤を見る / 料金を見る / 予約する）
- スクロールで体験が変わる上品なモーション（速すぎない・派手すぎない）

---

## 6. 機能要件

### 6.1 予約・問い合わせ導線

- ヘッダー固定CTA（電話 / 予約フォーム / Instagram誘導）
- 営業時間外は翌営業日の案内を表示
- 予約フォーム項目：来店希望日 / 来店時間帯 / 人数 / 初来店かどうか / 接待利用かどうか / 希望キャスト名（任意） / 連絡先

### 6.2 システム料金ページ

- セット料金・延長・指名・同伴・税・サービス料などを明確化
- ケース別の合計目安を表示
- ボトル価格は掲載しない

### 6.3 本日出勤表示と出勤表

- トップに本日出勤キャストを自動表示
- 出勤表は本日と週間の切替
- 管理画面でON/OFFと時間帯を編集可能

### 6.4 キャスト一覧と詳細（顔出し不可対応）

**一覧カード:** 写真（後ろ姿・雰囲気カット） / 源氏名 / 雰囲気タグ / 入店日・新人表示  
**詳細:** 写真ギャラリー / 雰囲気タグ / 接客タイプ / 趣味 / ひとこと / SNSリンク（任意）  
**検索・フィルタ:** 雰囲気（上品 / 落ち着き / 華やか / 会話好き / 聞き上手 / 盛り上げ） / 来店目的（接待向き / 同伴向き / 初来店向き）

### 6.5 ギャラリー

- シャンデリアを中心にした写真セクション
- 広い箱を伝えるワイド写真
- VIP席紹介
- 簡易のフロア切替 UI（可能なら）

### 6.6 イベント・ニュース

- 管理画面で投稿 / トップに最新3件 / 過去一覧ページ

### 6.7 アクセス強化

- Google Map / 最寄駅からのルート / タクシー用目的地テキストコピーボタン / 営業時間と定休日

### 6.8 FAQ

- 初来店向け不安解消 / 料金の仕組み / 予約の流れ / マナー・ルール

## 7. 差別化機能要件

### 優先度：高

#### 7.1 Night Style 診断（キャスト推薦）

- 3問程度の質問（接待 / 盛り上げ / 会話中心 / 初来店）
- 結果に応じてキャスト候補を提示
- 顔出し不可でも推薦体験で指名判断を支援

#### 7.2 本日の店内ムード表示

- 管理画面でムードを選択（落ち着き / 普通 / 賑やか）
- 当日の期待値調整

#### 7.3 お気に入りキャスト（端末保存）

- ログインなしでお気に入り登録
- 本日出勤時に強調表示
- サーバーに個人情報を持たない設計

### 優先度：中

#### 7.4 VIP席 空き状況簡易表示

- 三段階（空き / 残り少 / 満席）
- 接待層の予約判断を加速

#### 7.5 Voice Intro（任意）

- 30秒の音声自己紹介
- 顔出し不可の弱点を強みに変換
- 運用負荷が高い場合は後回し

---

## 8. デザインシステム要件

- 白を主役 / 余白は広く・線は細く・影は弱く
- タイポで格を出す / 写真が主役・装飾は最小限
- モーションは上品・遅め・控えめ

---

## 9. 非機能要件

- 表示速度最優先（画像最適化・遅延読み込み）
- スパム対策（フォーム）
- SEO基本（meta・構造化データ・店名・エリア・最寄駅）
- キャスト詳細は固有URLで生成

---

## 10. 管理画面要件

### 管理対象

- キャスト CRUD（写真・タグ・プロフィール・公開/非公開）
- 本日出勤 / 週間出勤
- イベント・ニュース / 本日のムード / VIP席空き状況 / ギャラリー画像

### 権限

- 管理者 / 編集者（店長想定）

---

## 11. 受け入れ基準

- トップでシャンデリアが象徴として成立している
- 白背景でも安っぽくならず、高級感が担保されている
- 顔出し不可でもキャスト選びが成立する
- 料金の不安が減り、予約導線が明確
- 管理画面で本日出勤とニュースが迷わず更新できる
- スマホで予約まで30秒以内に到達できる
