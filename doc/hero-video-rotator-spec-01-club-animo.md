# Club Animo Webサイト ヒーロー動画ローテーション設計書

0 目的
トップのヒーローで店内の空気感を最短で伝えるため、複数の店内動画を3秒ずつ自動再生し、上品なトランジションで順番に表示する。
高級感の維持を最優先とし、派手な演出や重い実装は避ける。
動画未設定時は静止画プレースホルダに自動フォールバックする。

1 ヒーロー動画の基本仕様

- 表示対象: トップページのHeroセクション
- 動画枚数: 3〜6本を推奨 (最小 1本, 最大 8本まで)
- 表示時間: 1本あたり3秒
- トランジション時間: 0.6秒〜0.9秒
- トランジション種類: fade, slide (管理画面で選択可能にする)
- ループ: 無限ループ
- 音声: 常にミュート (音声トラックがある場合もミュート固定)
- 再生: 自動再生, インライン再生 (OSの自動再生制限を考慮し muted + playsinline を必須)

2 動画ファイル要件

- 推奨フォーマット: mp4 h264 (可能ならwebmも用意し優先配信)
- 解像度: 1920x1080 推奨 (最小 1280x720)
- 長さ: 各動画 3〜8秒程度のループ向き素材 (長すぎる素材は禁止)
- 容量目安: 1本あたり 2MB〜6MB 目標 (最大でも 10MB を目安, 合計容量が重い場合は本数を減らす)
- 撮影内容: シャンデリア, フロアの広さ, VIP席, バーカウンター
- カメラワーク: ゆっくり, パンやドリーは最小限, 揺れの強い手ブレ禁止
- 色味: 明るい, 暖色照明, 白基調のUIに合う露出

3 表示レイヤー構造

- HeroContainer
- HeroMediaLayer: 動画 または 静止画を表示するレイヤー。複数動画を重ね、アクティブ1枚のみ表示する
- HeroOverlayLayer: 白背景UIとコピー可読性確保のための薄いオーバーレイ。濃くしすぎない。高級感を損なわない
- HeroCopyLayer: キャッチコピー, サブコピー
- HeroCtaLayer: 予約, 本日出勤, 求人

4 トランジション仕様
4.1 fade モード

- 現表示動画: opacity 1 -> 0
- 次動画: opacity 0 -> 1
- 同時に交差させる (タイミングは中央でクロスする)

  4.2 slide モード

- 方向: 右から左 を基本 (逆方向は禁止しないが統一が必要)
- 現表示動画: translateX 0 -> -3% か -5%, opacity 1 -> 0
- 次動画: translateX 3% か 5% -> 0, opacity 0 -> 1
- 移動量は小さく (大きいスライドは禁止)

5 コントロール仕様

- 自動再生: オン
- ユーザー操作: 基本は操作UIを出さない
- 例外: アクセシビリティを考慮し、停止ボタンをフッター近くまたは設定に用意できる拡張余地を残す
- reduced motion: OSがモーション軽減の場合、動画ローテーションを停止し、最初の動画を静止画として表示、または静止画ヒーローに切替

6 フォールバック仕様

- 動画0本: ヒーロー静止画プレースホルダを表示
- 動画読み込み失敗: 静止画に切替
- 通信が遅い場合: 最初の動画だけを表示してローテーション開始を遅延してもよい

7 パフォーマンス設計

- 初期表示では最初の動画だけを優先
- 2本目以降は遅延プリロード
- Next.jsでの基本方針: 動画は標準videoタグで扱う
- ポスター画像を必ず設定しLCPを悪化させない。各動画に紐づく poster を必須とし1MB以下推奨

8 管理画面運用

- 管理画面に Hero Media 管理を追加
- 項目: heroMediaMode, videoRotatorEnabled, transitionMode (fade or slide), durationMs (3000), transitionMs (700)
- 動画リスト: 並び順, 公開非公開, タイトル, 動画URL, posterURL

9 受け入れ基準

- トップヒーローで複数動画が3秒ごとに切り替わる
- fade と slide を切り替え可能
- ミュートかつインラインで自動再生される
- モバイルでもカクつきが目立たない
- 動画が無い場合は静止画プレースホルダに自動フォールバックする
- reduced motion でローテーションが停止する
