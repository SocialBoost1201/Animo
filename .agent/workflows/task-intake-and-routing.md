# Task Intake and Routing

## Purpose
新しい依頼を受けた時に、主担当エージェントと副担当の要否を決める。

## Steps
1. タスクを strategy / design / implementation / audit / debugging / documentation のどれかに分類する
2. 主担当を1つ選ぶ
3. 副担当が必要か判断する
4. ChatGPT に返す内容を整理する
   - 主担当
   - 副担当
   - 理由
   - 完了条件
   - 禁止事項

## Output
- 主担当
- 副担当
- 理由
- 次に渡す指示文の骨子
